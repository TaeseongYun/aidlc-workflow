# Adaptive Depth

작업의 복잡도에 맞게 워크플로우의 상세도를 조절한다.
단계(STEP)의 실행 여부는 `core-workflow.md`의 조건부 규칙이 결정하고,
이 문서는 실행되는 단계 안에서 **얼마나 깊이 다루는지**를 정의한다.

## 핵심 원칙

> 문제에 필요한 만큼만 상세하게, 그 이상도 이하도 아니게.

- 단순한 문제를 인위적으로 부풀리지 않는다.
- 복잡한 문제를 부족하게 다루지 않는다.
- 모든 필수 산출물은 depth와 무관하게 생성한다. depth는 내용의 상세도만 조절한다.

## Depth Level 정의

### minimal
- **대상**: 단순 버그 수정, 텍스트 변경, 기존 패턴 재사용, 명확한 1-2 파일 변경
- **특징**: 핵심 섹션만 작성, 조건부 STEP 대부분 스킵
- **질문 예산**: 3개/라운드

### standard
- **대상**: 중간 복잡도 기능 추가, 일부 설계 판단 필요, 2-5개 파일 영향
- **특징**: 표준 상세도, 조건에 따라 STEP 실행/스킵
- **질문 예산**: 7개/라운드

### comprehensive
- **대상**: 대규모 기능, 다수 도메인 영향, 신규 설계, 마이그레이션, 외부 연동
- **특징**: 전체 섹션 상세 작성, 조건부 STEP 대부분 실행
- **질문 예산**: 12개/라운드

## Depth Level 판정

### 판정 시점
STEP 1-A Discovery 완료 직후 (STEP 1-B).

### 판정 요소

| 요소 | minimal | standard | comprehensive |
|------|---------|----------|---------------|
| 요청 명확성 | 구체적, 1-2줄로 완결 | 의도 명확하나 상세 필요 | 모호하거나 다의적 |
| 영향 범위 | 단일 파일/컴포넌트 | 2-5개 파일/컴포넌트 | 다수 도메인, 시스템 전반 |
| 설계 결정 | 없음 (기존 패턴) | 1-3개 | 4개 이상 |
| 리스크 | 기존 패턴, 롤백 용이 | 일부 신규, 제한적 영향 | 신규 설계, 데이터 변경, 외부 연동 |
| 기존 CTX 커버리지 | 관련 규칙 있음 | 부분적 | 관련 규칙 없거나 부족 |

### 판정 규칙
- 5개 요소 중 3개 이상이 해당 depth에 맞으면 해당 level로 결정한다.
- 경계선인 경우 **한 단계 높은 depth**를 선택한다.
- 사용자가 명시적으로 depth를 지정하면 그것을 따른다.

### 기록
- `aidlc-state.md`의 `Depth Level` 필드에 기록한다.
- `audit.md`에 판정 사유를 기록한다.

## Depth별 산출물 상세도

### requirements.md

| 섹션 | minimal | standard | comprehensive |
|------|---------|----------|---------------|
| 목표 | 1-2줄 | 3-5줄 + 배경 | 상세 배경 + 제약 + 성공 지표 |
| In-Scope | 핵심 항목만 | 항목 + 간략 설명 | 항목 + 상세 설명 + 경계 조건 |
| Out-of-Scope | 1-2개 | 3-5개 | 상세 + 이유 |
| 기능 요구사항 | 핵심만 | 표준 | 전체 + 엣지 케이스 |
| 파생 요구사항 | 없음 또는 1-2개 | 있으면 작성 | 필수 |

### unit-of-work.md

| 항목 | minimal | standard | comprehensive |
|------|---------|----------|---------------|
| UOW 수 | 1-2개 | 3-5개 | 제한 없음 |
| 수용 기준 | 1-2개/UOW | 2-3개/UOW | 3개 이상/UOW + Gherkin |
| 의존성 | 간략 | 명시 | 상세 + 순환 검증 |

### 게이트 메시지

| 항목 | minimal | standard | comprehensive |
|------|---------|----------|---------------|
| 산출물 요약 | 2개 항목 | 3-4개 항목 | 5개 항목 |
| 파일 목록 | 핵심만 | 관련 전체 | 관련 전체 + 참조 |
