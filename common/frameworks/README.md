# PM Frameworks

선택적으로 사용할 수 있는 PM 프레임워크 모음이다.
planning-draft 또는 requirements 작성 시 필요에 따라 참조한다.

## 사용 원칙
- 프레임워크는 질문 추출과 우선순위 판단을 돕는 보조 도구다.
- team-ai-workflow의 거버넌스(추정 금지, approval gate)를 대체하지 않는다.
- 사용자가 특정 프레임워크를 요청하면 적용하고, 그렇지 않으면 자동 적용하지 않는다.

## 포함 프레임워크

| 파일 | 용도 | 적용 시점 |
|------|------|----------|
| `rice.md` | 기능 우선순위 평가 | 여러 feature가 경쟁할 때 |
| `jtbd.md` | 사용자 니즈 분석 | raw-request에서 진짜 목표를 찾을 때 |
| `moscow.md` | 범위 분류 | In-Scope / Out-of-Scope 정리 시 |
